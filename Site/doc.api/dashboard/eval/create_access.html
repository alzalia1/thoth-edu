/dashboard/eval/create_access

< retour
Enregistre un nouvel accès dans la base de donnée

Status

✅ Terminé

Requête

                    
{
    "name" : "nom de l'accès",
    "id_eval" : "id de l'eval associée à cet accès",
    "random" : true, // Rangement aléatoire des questions
    "time" : {
        "start" : "timestamp de début",
        "end" : "timestamp de fin"
    }
}

Réponse

                    
{
    "status" : "success ou fail",
    "reason" : "raison du fail ou none"
}
Remarque : reason peut soit contenir "none", soit ne pas être renvoyé du tout dans le cas d'un succès.

Exemple

                            
{
    "name" : "TG3",
    "id_eval" : "i7kao3",
    "random" : true,
    "time" : {
        "start" : "1720688400000",
        "end" : "1720684800000"
    }
}
→
                            
{
    "status" : "success",
}
Remarque : On a fait ici le choix de ne pas mettre reason vu que le status est "success".
